精选教程：创建你的第一个MCP服务

1. 准备工作

在创建MCP服务之前，确保你的系统满足以下要求：

  • 操作系统：Linux、macOS或Windows
  • Python：3.8或更高版本
  • Git：用于克隆MCP仓库
  • 网络连接：用于下载依赖和连接外部数据源
  • 推荐安装：Docker（用于容器化部署）

2. 安装MCP

首先，克隆Anthropic的MCP仓库并安装必要的依赖：

# 克隆MCP仓库
git clone https://github.com/anthropics/mcp.git
cd mcp

# 安装依赖
pip install -e .

# 安装开发依赖（可选）
pip install -e "[dev]"

3. 创建基本的MCP服务

现在，让我们创建一个基本的MCP服务。首先创建一个新的Python文件，命名为my_mcp_service.py：

from mcp.server import MCPServer, MCPService
from mcp.types import Message, Context

# 创建一个自定义的MCP服务类
class MyDataSource(MCPService):
    def __init__(self):
        super().__init__(name="my-data-source")
        # 注册服务方法
        self.register_method("query_data", self.query_data)
        
    async def query_data(self, context: Context, query: str) -> str:
        """查询数据的方法"""
        # 这里实现你的数据源查询逻辑
        # 例如，从数据库、API或本地文件中获取数据
        results = f"查询结果: {query}"
        return results

# 创建并启动MCP服务器
def main():
    # 创建服务器实例
    server = MCPServer(
        host="localhost",
        port=8080,
        # 可选的安全配置
        tls_cert=None,  # 生产环境中应提供TLS证书
        tls_key=None
    )
    
    # 添加我们的服务
    server.add_service(MyDataSource())
    
    # 启动服务器
    print("MCP服务器启动在 http://localhost:8080")
    server.start()

if __name__ == "__main__":
    main()

4. 配置MCP服务

为了使MCP服务更加安全和可靠，我们需要添加一些配置。创建一个配置文件mcp_config.json：

{
  "host": "localhost",
  "port": 8080,
  "log_level": "INFO",
  "cors_origins": ["*"],
  "auth": {
    "enabled": true,
    "api_keys": ["your-api-key-here"]
  },
  "tls": {
    "enabled": false,
    "cert_path": "",
    "key_path": ""
  }
}

然后修改我们的服务代码以使用这个配置文件：

import json
from mcp.server import MCPServer, MCPService
from mcp.types import Message, Context

# 创建一个自定义的MCP服务类
class MyDataSource(MCPService):
    def __init__(self):
        super().__init__(name="my-data-source")
        # 注册服务方法
        self.register_method("query_data", self.query_data)
        
    async def query_data(self, context: Context, query: str) -> str:
        """查询数据的方法"""
        # 这里实现你的数据源查询逻辑
        results = f"查询结果: {query}"
        return results

# 创建并启动MCP服务器
def main():
    # 加载配置文件
    with open("mcp_config.json", "r") as f:
        config = json.load(f)
    
    # 创建服务器实例
    server = MCPServer(
        host=config["host"],
        port=config["port"],
        log_level=config["log_level"],
        tls_cert=config["tls"]["cert_path"] if config["tls"]["enabled"] else None,
        tls_key=config["tls"]["key_path"] if config["tls"]["enabled"] else None,
        cors_origins=config["cors_origins"],
        api_keys=config["auth"]["api_keys"] if config["auth"]["enabled"] else None
    )
    
    # 添加我们的服务
    server.add_service(MyDataSource())
    
    # 启动服务器
    print(f"MCP服务器启动在 http://{config['host']}:{config['port']}")
    server.start()

if __name__ == "__main__":
    main()

5. 测试MCP服务

现在，让我们创建一个简单的客户端来测试我们的MCP服务。创建一个名为test_mcp_client.py的文件：

import asyncio
from mcp.client import MCPClient

async def main():
    # 创建MCP客户端
    client = MCPClient(
        server_url="http://localhost:8080",
        api_key="your-api-key-here"  # 如果启用了认证
    )
    
    try:
        # 连接到服务器
        await client.connect()
        
        # 调用服务方法
        result = await client.call_service(
            service_name="my-data-source",
            method_name="query_data",
            params={"query": "如何使用MCP协议？"}
        )
        
        # 打印结果
        print(f"服务返回结果: {result}")
        
    except Exception as e:
        print(f"发生错误: {e}")
    finally:
        # 关闭连接
        await client.disconnect()

if __name__ == "__main__":
    asyncio.run(main())

6. 运行MCP服务

现在，让我们运行我们的MCP服务并进行测试：

# 启动MCP服务器
python my_mcp_service.py

# 在另一个终端中运行客户端测试
python test_mcp_client.py

7. 部署MCP服务（使用Docker）

为了便于部署和扩展，我们可以使用Docker来容器化我们的MCP服务。首先创建一个Dockerfile：

# 使用Python官方镜像作为基础镜像
FROM python:3.10-slim

# 设置工作目录
WORKDIR /app

# 复制项目文件到容器中
COPY . .

# 安装依赖
RUN pip install --no-cache-dir -e .

# 暴露端口
EXPOSE 8080

# 运行服务
CMD ["python", "my_mcp_service.py"]

然后创建一个docker-compose.yml文件以简化部署：

version: '3'

services:
  mcp-service:
    build: .
    ports:
      - "8080:8080"
    volumes:
      - .:/app
    environment:
      - MCP_CONFIG=mcp_config.json
    restart: unless-stopped

使用Docker Compose启动服务：

docker-compose up -d

8. 高级功能：添加流式响应

MCP支持流式响应，这对于处理大量数据或长时间运行的任务非常有用。让我们修改我们的服务以支持流式响应：

from mcp.server import MCPServer, MCPService
from mcp.types import Message, Context, Stream
import asyncio

# 创建一个支持流式响应的MCP服务类
class MyStreamingDataSource(MCPService):
    def __init__(self):
        super().__init__(name="my-streaming-data-source")
        # 注册服务方法
        self.register_method("stream_data", self.stream_data)
        
    async def stream_data(self, context: Context, query: str) -> Stream[str]:
        """流式返回数据的方法"""
        # 模拟流式返回数据
        for i in range(5):
            # 生成部分结果
            partial_result = f"部分结果 {i+1}: {query}"
            # 使用yield返回流式数据
            yield partial_result
            # 模拟处理延迟
            await asyncio.sleep(1)

# 在main函数中添加这个服务
# server.add_service(MyStreamingDataSource())

9. 监控与日志

为了确保MCP服务的稳定运行，我们需要添加监控和日志功能。MCP提供了内置的日志功能，我们可以通过配置文件调整日志级别：

# 修改mcp_config.json中的log_level字段
# 可选值: DEBUG, INFO, WARNING, ERROR, CRITICAL
"log_level": "INFO"

10. 生产环境部署建议

在生产环境中部署MCP服务时，建议采取以下措施：

  1. 启用TLS加密，使用有效的SSL证书
  2. 配置API密钥认证，定期轮换密钥
  3. 限制CORS源，只允许受信任的域名访问
  4. 使用反向代理（如Nginx）来处理请求路由和负载均衡
  5. 设置适当的资源限制（CPU、内存）
  6. 配置日志聚合和监控系统
  7. 实现自动备份和故障恢复机制

常见问题

解答在使用MCP过程中可能遇到的常见问题

MCP（Model Context Protocol）是由Anthropic推出的开源协议，旨在实现大语言模型与外部数据源和工具的安全集成。它的主要用途是在大模型和外部系统之间建立一个安全、可靠的双向通信桥梁，使模型能够访问和操作外部数据和服务，同时保持安全性和隐私性。

与传统的API集成方式相比，MCP提供了更高级别的安全机制和更灵活的交互模式。它支持流式数据传输、上下文保留和丰富的类型系统，可以更好地满足大语言模型的特殊需求。此外，MCP还提供了标准化的错误处理和认证机制，使集成过程更加规范和可靠。

确保MCP服务安全的方法包括：启用TLS加密、配置API密钥认证、限制CORS源、使用防火墙限制访问、定期更新依赖库、实现访问日志和监控系统、定期轮换凭证和密钥，以及遵循最小权限原则设计服务权限。在生产环境中，建议使用反向代理和负载均衡器来增强安全性。
