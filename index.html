Featured Tutorial: Creating Your First MCP Service

1. Preparation

Before creating an MCP service, ensure your system meets the following requirements:

  • Operating System: Linux, macOS, or Windows
  • Python：3.10或更高版本', 'req-network': '网络连接：用于下载依赖和连接外部数据源',
  • Git
  • Network connection: for downloading dependencies and connecting to external data sources
  • Recommended: Docker (for containerized deployment)

2. Installing MCP

First, install the MCP package using pip:

# Install MCP package
pip install mcp

# Install development dependencies (optional)
pip install "mcp[cli]"

3. Creating a Basic MCP Service

Now, let's create a basic MCP service. First, create a new Python file called my_mcp_service.py:

from mcp.server.fastmcp import FastMCP

# Create an MCP server
mcp = FastMCP("Demo")


# Add an addition tool
@mcp.tool()
def add(a: int, b: int) -> int:
    """Add two numbers"""
    return a + b


# Add a dynamic greeting resource
@mcp.resource("greeting://{name}")
def get_greeting(name: str) -> str:
    """Get a personalized greeting"""
    return f"Hello, {name}!"


# Add a prompt
@mcp.prompt()
def greet_user(name: str, style: str = "friendly") -> str:
    """Generate a greeting prompt"""
    styles = {
        "friendly": "Please write a warm, friendly greeting",
        "formal": "Please write a formal, professional greeting",
        "casual": "Please write a casual, relaxed greeting",
    }

    return f"{styles.get(style, styles['friendly'])} for someone named {name}."

4. start MCP Service

To make your MCP service more secure and reliable, we need to add some configuration. Create a configuration file called mcp_config.json:

{
  "Demo": {
    "command": "npx",
    "args": [
      "-y",
      "@upstash/context7-mcp"
    ],
    "env": {},
    "working_directory": null
  }
}

Congratulations! You have successfully created and deployed an MCP service. Through this service, you can securely integrate external data sources with large language models, providing richer functionality and data support for your AI applications.

Frequently Asked Questions

Answers to common questions you may encounter when using MCP

MCP (Model Context Protocol) is an open-source protocol for secure integration between large language models and external data sources and tools. Its main purpose is to establish a secure, reliable two-way communication bridge between large models and external systems, enabling models to access and operate external data and services while maintaining security and privacy.',

Compared to traditional API integration methods, MCP provides higher-level security mechanisms and more flexible interaction patterns. It supports streaming data transmission, context preservation, and a rich type system, which can better meet the special needs of large language models. Additionally, MCP provides standardized error handling and authentication mechanisms, making the integration process more standardized and reliable.

Methods to ensure the security of MCP services include: enabling TLS encryption, configuring API key authentication, restricting CORS origins, using firewalls to limit access, regularly updating dependency libraries, implementing access logs and monitoring systems, regularly rotating credentials and keys, and designing service permissions according to the principle of least privilege. In production environments, it is recommended to use reverse proxies and load balancers to enhance security.
